# План включения маркетплейса (после открытия ИП)

Сейчас Progressly — «личная школа» самозанятого: это требование закона и ЮKassa.
Вся архитектура уже готова к маркетплейсу — включение займёт дни, а не месяцы.

## Что уже заложено (ничего переделывать не придётся)

- **Каталог в БД** (`catalog_courses`): новый курс любой школы = новая строка
  (название, цена, автор, `commission_pct`). Витрина и оплата подхватят сами.
- **Контент по slug** (`course_modules` / `course_lessons`): курсы любых авторов
  живут рядом, доступ режется RLS по `course_access`.
- **Оплата**: `create-payment` берёт цену из БД; в `course_orders` уже есть
  `author_id`, `commission_pct`, `commission_amount` — история продаж сразу
  пригодна для расчёта выплат авторам.
- **Флаг** `app_settings.marketplace_mode` — главный выключатель режима.
- **AI-репетитор и прогресс** работают с любым курсом по slug.

## Чек-лист включения (по порядку)

1. **Юридическое**: открыть ИП (УСН 6 %, ОКВЭД 85.41 + 63.11), подать новую
   заявку в ЮKassa уже как ИП, получить новые ключи магазина →
   `supabase secrets set YOOKASSA_SHOP_ID=... YOOKASSA_SECRET_KEY=...`.
   Обновить оферту и контакты: исполнитель — ИП, добавить агентскую схему
   (платформа действует по поручению авторов курсов).
2. **Флаг**: в SQL-редакторе
   `update app_settings set value = 'true' where key = 'marketplace_mode';`
3. **Регистрация авторов**: вернуть в `AuthScreen.jsx` роли «Учитель» и
   «Школа» (код сохранился в git-истории, коммит `e701864` — там был выбор
   из 3 ролей). Показать выбор ролей только при `marketplace_mode = true`.
4. **Кабинет автора**: вернуть из git-истории `TeacherCabinet` (удалён в
   `e701864`) и подключить к реальным данным: его курсы из `catalog_courses
   where author_id = uid`, продажи из `course_orders`, выплаты = сумма
   `amount - commission_amount` за период.
5. **Загрузка чужих курсов**: для каждого нового курса — строка в
   `catalog_courses` (+ `author_id`, `commission_pct` 15–20) и контент в
   `course_modules`/`course_lessons` через сид-скрипт
   (`scripts/seed_informatics.mjs` — скопировать и поменять slug/файлы).
6. **Вебхук оплаты**: в `yookassa-webhook` при выдаче доступа заполнять
   `commission_amount = amount * commission_pct / 100` (одна строка кода).
7. **Витрина**: при `marketplace_mode = true` показывать на карточках
   `school_name`/`teacher_name` из БД (уже показываются) и вернуть фильтры
   по предметам (код в истории git, коммит `e701864`).
8. **Выплаты авторам**: первые месяцы — вручную раз в месяц по отчёту
   (`select author_id, sum(amount - commission_amount) from course_orders
   where status='succeeded' group by 1`); автоматизация — потом.

## Что НЕ делать до ИП

- Не включать `marketplace_mode` и регистрацию учителей.
- Не размещать чужие курсы и не принимать за них деньги — это нарушение
  режима самозанятости и причина первого отказа ЮKassa.
- Витрина должна оставаться «авторские курсы Кирилла Шевелева».

## Как поменять цену курса (уже сейчас, без деплоя)

Supabase → Table Editor → `catalog_courses` → поле `price`.
Витрина и платёж подхватят новую цену сразу (кэш каталога — до перезагрузки страницы).
