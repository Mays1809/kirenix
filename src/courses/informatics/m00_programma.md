# 💻 ИНФОРМАТИКА ЕГЭ 2027 — Полный курс с нуля до 90+

> **Расширенное издание курса Kirenix.**
> Подготовка к ЕГЭ по информатике для 10–11 классов.
> Язык программирования — **Python 3**. Всё ПО — **импортозамещённое** (LibreOffice).
> Курс построен так, чтобы ученик мог разобраться **с нуля и без учителя**.

## 📌 Как устроен этот курс

Курс разбит на **11 модулей** (отдельные файлы) + эта программа + финальная подготовка. Каждый модуль посвящён группе тем и заданий ЕГЭ. Внутри модуля — уроки. Каждый урок построен по единой строгой структуре:

| Блок | Что внутри |
|------|-----------|
| 🎯 **Что изучим** | список целей урока |
| 📖 **Теория** | подробное объяснение «почему», вывод формул, схемы, таблицы |
| 💡 **Разобранные примеры** | 6–10 примеров от простого к сложному, включая реальные задачи ЕГЭ и демо-2027, с полным пошаговым решением |
| 🧩 **Все типы этого задания** | (для крупных заданий) разные формулировки одного номера ЕГЭ |
| ✍️ **Практика** | 15–20 заданий для самостоятельного решения |
| ✅ **Ответы к практике** | в конце каждого урока |
| 📝 **Шпаргалка** | сжатый конспект формул и алгоритмов для повторения |
| ⚠️ **Типичные ошибки** | с примерами, как делать НЕ надо |
| 💻 **Код на Python** | где уместно — полный рабочий код |

**Два уровня сложности:**
- 🟢 **Базовый (0 → 70 баллов)** — модули 1–6. Закрывает ~20 первых заданий.
- 🔵 **Продвинутый (70 → 90+ баллов)** — модули 7–11. Сложные задания: теория игр, динамическое программирование, задания 22–27.

## ⚡ ИЗМЕНЕНИЯ ЕГЭ 2027 (проверено по ФИПИ и свежим источникам)

Все изменения **учтены** в курсе. Главное, что важно знать выпускнику 2027 года:

**1. Задание 12 — новый исполнитель «Машина Тьюринга».**
Раньше в задании 12 был только исполнитель «Редактор» (работа со строкой). В 2027 году в части вариантов его заменяет **Машина Тьюринга** — абстрактный исполнитель с бесконечной лентой, головкой и таблицей команд. Условие громоздкое (алгоритм расписан на 1–2 страницы), задание решается **только вручную** (не программой). Это объективно сложнее прежнего формата. Разбираем **оба** варианта (Модуль 6, уроки 6.2 и 6.3).

**2. Задание 22 — новый тип «на максимум».**
Тема та же — параллельные (многопроцессорные) вычисления с зависимостями между процессами. Но теперь спрашивают **максимальное** время одновременного выполнения процессов / **максимальное число** процессов, которые успеют завершиться к заданному моменту (раньше искали минимальное время). Меняется логика рассуждения. Разбираем в Модуле 10.

**3. Форматы файлов — импортозамещение.**
Из-за перехода на российское/открытое ПО файлы заданий теперь в открытых форматах LibreOffice:

| Формат | Задания | Программа |
|--------|---------|-----------|
| `.ods` (таблицы) | **3, 9, 18, 22** | LibreOffice Calc |
| `.odt` (документ) | **10** | LibreOffice Writer |
| `.txt` (текст) | **17, 24, 26, 27** | любой редактор / Python |

Формулы LibreOffice Calc почти полностью совпадают с Excel, но **разделитель аргументов — точка с запятой `;`**, а в части локалей — особенности (`СЧЁТЕСЛИ`, `СУММЕСЛИ` и т.д.). Учтено в Модуле 5.

**4. Структура КИМ не изменилась.**
Всего **27 заданий**: **11 базового** + **11 повышенного** + **5 высокого** уровня. Время — **3 часа 55 минут (235 минут)**. Все ответы — короткие (число, набор цифр или слово). Часть заданий выполняется на компьютере с реальными файлами.

## 🎓 КАК ЗАНИМАТЬСЯ ПО КУРСУ (методика)

1. **Иди по порядку модулей.** Базовый уровень — фундамент. Не перепрыгивай к теории игр, пока не освоил Python (Модуль 3).
2. **Сначала теория, потом примеры, потом практика.** Не подглядывай в ответы практики, пока не решишь сам.
3. **Разобранные примеры — не читай, а прорешивай.** Закрой решение и попробуй сделать сам, потом сверься.
4. **Программируемые задания (17, 24, 25, 26, 27) учись решать кодом.** На реальном ЕГЭ у тебя будет компьютер с Python — это твоё преимущество.
5. **Заданий 9 и 22 — тренируйся прямо в LibreOffice Calc** с файлом `.ods`.
6. **Шпаргалку из каждого урока** выпиши от руки — это лучшее повторение перед экзаменом.
7. **Раз в 2 недели** решай полный пробник на время (235 минут) — вырабатывай выносливость.

**Ориентир по баллам** (информатика — 27 заданий, максимум **29 первичных**: задания 1–25 по 1 баллу, 26 и 27 — по 2 балла):
- Задания **1–25** (по 1 баллу) — это до **25 первичных ≈ 90 тестовых**.
- Все **29 первичных** (вся первая часть + задания 26 и 27) = **100 тестовых**.
- Для медали нужно **70+ тестовых** — достаточно уверенно закрыть бо́льшую часть заданий 1–25.
