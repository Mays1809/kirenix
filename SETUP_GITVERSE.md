# Перенос сайта на GitVerse Pages (РФ, без VPN, бесплатно)

GitHub Pages в РФ режет РКН → сайт открывается только с VPN, из-за чего виснет
ЮKassa. Решение — публиковать тот же статический сайт на **GitVerse Pages**
(СберТех, работает в РФ без VPN, бесплатно). Код уже готов:
`.gitverse/workflows/pages.yml` (сборка Vite → публикация) и `public/.nojekyll`.

## Шаги (делает Кирилл)

1. **Регистрация.** Зайди на https://gitverse.ru, войди по **Sber ID** (бесплатно).

2. **Репозиторий.** Создай **публичный** репозиторий с именем, равным имени твоего
   аккаунта (для сайта на корне адреса). Пример: аккаунт `kirenix` → репозиторий
   `kirenix` → сайт будет на `https://kirenix.gitverse.site`.
   В настройках репозитория **включи CI/CD**.

3. **Залей код** (второй remote, GitHub остаётся как есть):
   ```
   cd ~/kirenix
   git remote add gitverse https://gitverse.ru/<твой_аккаунт>/kirenix.git
   git push gitverse main
   ```
   Workflow `.gitverse/workflows/pages.yml` уедет вместе с пушем.

4. **Включи Pages.** Репозиторий → **Настройки → Страницы**:
   - включи функциональность;
   - **Источник = «Воркфлоу»**;
   - запусти workflow (или он стартует сам после пуша).
   Дождись зелёной сборки → сайт на `https://<аккаунт>.gitverse.site`.

5. **Проверь без VPN:** открой адрес, проверь вход, курсы и **оплату ЮKassa**
   (она оживёт, потому что ты больше не под VPN).

6. **Свой домен `kirenix.ru`.** В **Настройки → Страницы** поищи поле кастомного
   домена (GitVerse обещали поддержку CNAME в Q1 2026 — возможно, уже доступно):
   - если есть → добавь `kirenix.ru`, затем в reg.ru пропиши DNS, как укажет GitVerse;
   - если ещё нет → пока работаем на `*.gitverse.site`, домен прицепим позже.

## Заметки
- Сборка идёт на серверах GitVerse (`npm ci && npm run build`), секреты не нужны —
  адрес Supabase и публичный ключ уже зашиты в коде.
- GitHub можно оставить как резервную копию; когда `kirenix.ru` будет указывать на
  GitVerse, старый GitHub Pages можно отключить.
- Если после переноса вход иногда подвисает — это уже про Supabase (Cloudflare) в РФ,
  лечится повтором запроса (уже встроен) — напиши, разберёмся отдельно.
