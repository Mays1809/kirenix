# Запуск продаж курса — пошаговая инструкция

Код уже в репозитории. Чтобы всё заработало, нужно выполнить 6 шагов
(минут 30, из них больше всего — регистрация в ЮKassa).

---

## Шаг 1. Применить миграцию БД

Вариант А (Supabase CLI, из корня проекта):
```bash
supabase db push
```

Вариант Б (без CLI): открой Supabase Dashboard → SQL Editor → вставь содержимое
`supabase/migrations/20260610200000_course_paywall.sql` → Run.

Что создаётся: таблицы `course_modules`, `course_lessons` (тела уроков под RLS),
`course_access` (доступ выдаёт только сервер), `course_orders`, функция
`has_course_access()`, публичное оглавление `course_lessons_index`.
Заодно закрывается дыра самозаписи в `enrollments` и твоему аккаунту
(maysbad@gmail.com) выдаётся роль admin + доступ к курсу.

> ⚠️ Если ты ещё ни разу не входил на сайт под maysbad@gmail.com —
> сначала зарегистрируйся, потом выполни последние два блока миграции повторно.

## Шаг 2. Загрузить контент курса в БД

Возьми сервисный ключ: Dashboard → Settings → API → `service_role` (secret).

```bash
SUPABASE_SERVICE_ROLE_KEY="eyJ..." node scripts/seed_informatics.mjs
```

Скрипт зальёт 11 модулей и 49 записей (38 уроков + 11 итогов), урок 1.1 пометит
бесплатным. Запускать можно повторно — контент просто обновится.
Сервисный ключ никуда не сохраняй и не коммить.

## Шаг 3. Зарегистрировать магазин ЮKassa (для самозанятых)

1. yookassa.ru → «Подключить» → выбери режим **самозанятый** (понадобится
   подтверждение статуса НПД — ЮKassa проверит через «Мой налог»).
2. После активации: Личный кабинет → Интеграция → Ключи API:
   скопируй **shopId** и **Секретный ключ**.
3. Чеки выбивать не нужно — для самозанятых ЮKassa сама передаёт доход
   в «Мой налог».

## Шаг 4. Прописать секреты в Supabase

```bash
supabase secrets set \
  YOOKASSA_SHOP_ID="123456" \
  YOOKASSA_SECRET_KEY="live_..." \
  SITE_URL="https://ТВОЙ-ДОМЕН.vercel.app"
```

`SITE_URL` — адрес сайта, на который ЮKassa вернёт покупателя после оплаты.

## Шаг 5. Задеплоить функции оплаты

```bash
supabase functions deploy create-payment
supabase functions deploy yookassa-webhook --no-verify-jwt
```

(`--no-verify-jwt` обязателен для вебхука — он приходит от ЮKassa, а не от юзера;
безопасность обеспечивается перепроверкой платежа напрямую в API ЮKassa.)

Затем в ЛК ЮKassa → Интеграция → HTTP-уведомления укажи URL:
```
https://bgidvsfjnpitiosiwjar.supabase.co/functions/v1/yookassa-webhook
```
и отметь события **payment.succeeded** и **payment.canceled**.

## Шаг 6. Задеплоить сайт

```bash
git push
```

## Как это работает

- **Витрина** (модули, названия уроков, программа) — публичная.
- **Урок 1.1 — бесплатный** для всех (поднимает конверсию).
- **Тела остальных уроков** сервер отдаёт только купившим или админу — это RLS
  в Postgres, обойти из браузера нельзя; в JS-бандле контента больше нет.
- **Покупка**: кнопка → Edge Function `create-payment` (цена задана на сервере:
  5200 ₽) → страница ЮKassa → вебхук перепроверяет платёж и выдаёт доступ
  записью в `course_access` → покупатель возвращается на сайт, курс открыт.
- **Самозапись невозможна**: insert в `course_access`/`course_orders` доступен
  только сервисной роли.

## Проверка перед запуском

1. В ЛК ЮKassa включи **тестовый магазин**, возьми тестовые ключи, временно
   поставь их в секреты (шаг 4) и сделай покупку тестовой картой
   `5555 5555 5555 4477` (любые CVC/срок).
2. Убедись: после оплаты курс открылся, в `course_orders` статус `succeeded`,
   в `course_access` появилась запись.
3. Верни боевые ключи.

## Поменять цену

В двух местах (должны совпадать):
- сервер: `supabase/functions/create-payment/index.ts` → `COURSES.informatics.amount`
- витрина: `src/Platform.jsx` → `COURSES[0].price`

После правки функции: `supabase functions deploy create-payment`.
